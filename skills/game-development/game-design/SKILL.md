# Game Design

Game design patterns. Core loops, progression, economy, level design, player psychology.

## Instructions

### Core Loop

Short (30s to 3 min), satisfying, repeatable.
Action -> Feedback -> Reward -> Repeat.

### Progression

Linear: one path (story games).
Skill tree: spend points on abilities (RPGs).
Prestige: reset for permanent bonus (idle games).

### Economy Design

Faucets (sources): level rewards, daily login, achievements.
Sinks (drains): upgrades, cosmetics, energy refills.
Sink rate must match or exceed faucet rate to prevent inflation.
Soft currency: easy to earn, spent on progression.
Hard currency: rare or paid, spent on premium items.

### Difficulty Tuning

Flow state: challenge slightly above skill level.
Curve: tutorial easy, then ramp, spike, relief, harder ramp.
Dynamic difficulty: adjust based on player performance.

### Level Design

Teach through play: first encounter of mechanic must be safe to fail.
Landmarks: unique visuals aid navigation.
Pacing: alternate intense and calm sections.
Reward exploration: secrets for curious players.

### Reward Schedules

Fixed ratio: reward every N actions (tutorials).
Variable ratio: reward randomly (most engaging, loot boxes).
Fixed interval: reward every N minutes (daily login).

### Player Types (Bartle)

Achievers: win, complete, collect.
Explorers: discover secrets and lore.
Socializers: interact with others.
Killers: compete and dominate.
